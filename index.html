Developer quickstart — from zero to sample in ~3 minutes

Plain-English summary: request a sandbox key, POST an invoice, handle webhook for status updates. Expand code samples for copy-paste snippets.

Quickstart (3-minute)

  1. Request sandbox key — sign up and request a sandbox key (or use the demo key shown in the playground).
  2. POST an invoice — call POST /v1/invoices with invoice payload.
  3. Receive NRS confirmation — TaxBridge validates and forwards to NRS; you receive webhook events.

Note: Replace DEMO_SANDBOX_KEY with your real key in production.

Webhook handler (Node / Express) — signature verification

// Example: verify X-TaxBridge-Signature header (HMAC SHA256)
const express = require('express');
const crypto = require('crypto');
const app = express();
app.use(express.raw({type:'application/json'}));
app.post('/webhooks/taxbridge', (req, res) => {
  const sig = req.headers['x-taxbridge-signature'] || '';
  const secret = process.env.WEBHOOK_SECRET;
  const h = crypto.createHmac('sha256', secret).update(req.body).digest('hex');
  if (!crypto.timingSafeEqual(Buffer.from(h), Buffer.from(sig))) return res.status(401).send('invalid');
  const payload = JSON.parse(req.body.toString());
  // handle payload
  res.sendStatus(200);
});

Deployment & Release Checklist

Recommended stack

Next.js static export or single-file static SSG. Serve via CDN (Cloudflare / Fastly) with edge caching and image optimization. Build: npm run build && npm run export.

Performance hints

  • AVIF/WebP images, lazy-load hero assets
  • Compress JS bundles, target ≤200KB gzipped
  • Use CDN for static assets; set long cache-control and revalidate on deploy
